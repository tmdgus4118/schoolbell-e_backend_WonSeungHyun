3번 문제 풀이 과정입니다.

문제 개요
  - MySQL 또는 PostgreSQL을 사용하여 여러 단계의 승인 및 반려가 가능한 결재 시스템을 구축하는 문제입니다.
  - 이 시스템에서는 최소한의 테이블을 정의하고, 특정 사용자가 처리해야 할 결재 건을 조회하는 SQL 쿼리를 작성해야 합니다.

2. 해결 접근 방식
  - 결재 요청과 상태 관리 (승인/반려 가능)
  - 결재 단계별 담당자 지정 (다단계 결재 가능)
  - 특정 사용자가 처리해야 할 결재 건 조회 기능

* 테이블 설계 *
 - 최소한의 테이블을 사용하여 결재 시스템을 구축하기 위해 아래와 같은 3개 테이블을 정의하였습니다.


 - 1️⃣ users (사용자 테이블)
 - 2️⃣ approval_requests (결재 요청 테이블)
 - 3️⃣ approval_steps (결재 단계 테이블)

결론

✅ 최소한의 테이블(users, approval_requests, approval_steps)로 결재 시스템 설계✅ 승인 및 반려 상태 관리 가능 (pending, approved, rejected)✅ 특정 사용자가 처리해야 할 결재 건을 효율적으로 조회하는 SQL Query 작성
